The Project
A System Calender App

Calender is a pre-installed calender app that comes with a new dual-screen phone system called Pear. Users can plan, view and edit their schedules and also sync with other apps. The app provides a monthly, weekly, and daily view of the schedule, and can be used on a single phone screen and/or a dual screen view.

Role & Team

Team Size: 5

My Role: UI/UX Designer, mainly focused on the agenda view and daily view.

Others' Roles: Designers

Tools Used

Design Tools: Figma, Adobe Illustrator

Researches: Competetor Analysis

Competitor Analysis
Other Calender Apps Out There

We started by researching other calender apps on the market. We named calender apps we've used and/or heard of, and also searched in the Apple Store and Google Play for apps we haven't heard of.

We saw some really fancy apps, like Timepage and Vantage Calendar. These apps have a really smooth interaction and awesome visual effects. But they can be hard to use for beginners.

Other apps we tested were more basic and easy to understand. Like Apple Calender, Google Calender and Fantastical. These apps provided the basic set of functions that users would need, but in different ways.

Constrains
Dive Deep into Design System

Our app was designed based on a design language called Pear. Pear has its own style guide and design system guidelines that we needed to follow.

The top panel, the font (and font size), and the color palette are provided in the design system.

We also saw the list and card view that might be useful in our design.

Brain Storming
Design Goal

Our app is a pre-installed app. So unlike the fancy apps we saw, our user group was people that purchase the Pear phone. Since this is a new, cutting edge phone, our users would be younger people that love new technology, or just want to be different/trendy.

To make the app useful for all the users, we didn't want it to be fancy. The app would be pretty basic, easy to start using, and cover all basic needs. Which means users should be able to add, view, and edit events, see and organize schedules, and get notices.

We also wanted to make sure the flow and interaction of this app are smooth and efficient. When testing other apps, we realized some of them were harder to use becuase the flow was not well designed. And this is the main focus of our design, to offer basic functions in the easiest way.

Ideation
Route We Take

We narrowed down the screens we needed by asking the question "is this necessary?". And with the group of basic functions we want to acheive, we narrowed down to 4 basic screens.

A monthly view, a weekly view, a daily view, and Add Event. Edit Event and View Event would be similar to add event.

And monthly and weekly views are presented in "agenda view", which all the events are in the same size and more than one day can be shown.

Daily view is presented with timelines so the users would be aware of when they are free, and when they are not.

Problem and Solution
Navigation Between Views

The first problem we encountered is how to navigate between all the different views we have. We thought about a navigation bar, a hidden side bar and some other "bar" navigations.

These would work, but they also took up a lot of space and we didn't want to make it over-crowded. Also, things like hidden bars did not exist in the system design guideline.

So I proposed that we could try to include swipe. A lot calender apps we saw used swipe as their main way of interaction. And the design wouldn't be too complicated if we only use a little.

Result
The Calender

The Schedule
Agenda View

Agenda View offers a neat view for the users to see/organize their schedule. All days and events can be shown here, depending on the amount.

Users don't need to jump between days to see everything, so this will be really helpful for organizing and planning tasks, to-dos and due dates.

The Detail
Daily View

To see more details, like how busy and full the schedule is, when the user is free, and time gaps between events. We have to use the daily view.

In the daily view, the size (length) represents the duration of the event. Events are placed on the timelines so the users can read easily.

Month / Week / Day
Navigation

Hold and drag the week bar down, the week would expand into a month. And hold the month bar and swipe up, the month would shrink into a week.

The navigation between the agenda view and the daily view is done by tapping the date. Tap a date on the agenda view, the agenda would expand to form the daily view. Tap the date on the daily view, the daily view would shrink to form the agenda view.

Interaction
Adding Event

The Add Event screen takes a list of parameters, but only title and time(duration) are required.

The users can also add more details if they want, like location, repeat, reminder, attendee, and note.
