The Project
An Online Gaming Community

Arcode is a gaming website and community for young people that offers three games, a registration system, a leaderboard and a comment community. Users can enjoy games, keep track of their performance and compete with other users. They can also communicate with each other and/or the developer team through the comment section to make connections and provide advices and critiques.

Role & Team

Team Size: 4

My Role: UI/UX Designer & Developer

Others' Roles: Game Developer, Database Developer, System Developer

Tools Used

Design Tools: Figma, Adobe Illustrator

Development Tools: HTML, CSS, Javascript, React, Google Firebase

Web Design
Design Draft

Our team selected two color palette to start the process. I did a small-scale Competetor Analysis and examined a lot game websites for inspiration.

The designing goal of the homepage is to showcase the games and attract the user. And the design should also be easy-understanding and the user should be able to find the games (and other resources) very easily.

I created two different design drafts using the two color palettes. Then we discussed and voted, and decided to go with the right one. Beacuse it looked more young and fun, which fitted better with our subject and our target user groups.

Front-end Development
Building the Web

The website is developed using a JavaScript library called React. We made the components and then rendered them all together to create the website. I built the interface for the homepage as I designed.

My teammates built the games we need, and also a database using Google Firebase to store users' data.

Design & Development
Comment Section & Leaderboard

After the games and the database were ready, we registered a sample account and tested/played with the website to see how should we design the comment section and the leaderboard.

Considering the time and stress on the development part, we decided to utilize the resources we have, start with basic and functioning designs and then make improvements.

With our basic comment section ready, I redesigned the look slightly, applied the color palette and typography so it fitted the overall theme without changing too many lines of code.

For the leaderboard, we were able to find a library that solves our problem perfectly. After applying our design language, we had a beautiful leaderboard that did exactly what we wanted.

Graphic Design
Game Cover

My next task is to design a game cover for each game. These game covers should not be too complicated, but they should be linked to the real game (and the website) closely. I designed them in three different ways.

The first game, Reacteroid, has a really beautiful interface. I used a screenshot that captured all its elements as the background, and used the website's main font and text color to highlight the name.

For SNAKE, I used its background color as the background, recreated its core elements and used the main colors of the game to color the text.

Fifteen Puzzle didn't have a fixed background, so I used the wesite color palette instead, and created a sample illustration to capture the core elements of this game.

Quick Challenge
Site Description

To fix a problem we have on the carousel (the biggest part on the homepage), I decided to make the site descriptions into graphics. Since we have three paragraphs, I thought pairing each paragraph with a game would be instreresting.

I kept the elements of the game covers, but left a lot white space to place the texts. I tested the font size on many screen sizes and finalized the designs.
